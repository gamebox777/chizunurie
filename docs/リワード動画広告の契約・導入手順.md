# リワード動画広告の契約・導入手順（Google 広告）

白地図塗りゲーム（chizunurie）の Web アプリに、**動画視聴で塗りポイントを回復するリワード動画広告**を
実際に導入するための契約・設定手順をまとめる。前提として AdSense アカウントは保有済み。

関連: [マネタイズ方針.md](マネタイズ方針.md) / 実装済みのモック: [Map.tsx](../frontend/src/components/Map.tsx) ・ [routes/points.ts](../backend/src/routes/points.ts)

## 結論

- **AdSense アカウントだけでは Web のリワード動画は出せない**（AdSense はディスプレイ広告のみ）。
- **AdMob はアプリ専用 SDK**なので、Web アプリでは動かない。
- **Web でリワード動画を正規配信できる Google 商品は Google Ad Manager (GAM) だけ**。
  AdSense はそこに「需要（広告の出し手）」として接続する役回りになる。

## 全体像

| 商品 | リワード動画 | 今回の Web アプリで使えるか |
|---|---|---|
| **AdSense**（保有済み） | Web サイトでは**非対応**（ディスプレイのみ） | 単独では✕。GAM の需要ソースとして連携 |
| **AdMob** | 対応 | ✕（**アプリ専用 SDK**。Web では動かない） |
| **Google Ad Manager** | **Web 対応**（PC / スマホ / タブレット） | ◎ これが本命 |

GAM には無料版（旧 DFP Small Business）と GAM 360（有償・大規模向け）があり、**まずは無料版で開始できる**。

## 契約・導入のステップ

1. **Google Ad Manager に登録**
   - AdSense と**同じ Google アカウント**で https://admanager.google.com にサインアップ。
   - サイト所有・住所・支払いプロファイルを設定。

2. **AdSense を需要ソースとして連携**
   - GAM 管理画面 → 管理 → リンクされたアカウント（Linked accounts）で AdSense を接続。
   - これで AdSense の広告需要がリワード枠の入札に回る。
   - 余力があれば **Ad Exchange (AdX)** も接続するとフィル率が上がるが、AdX は審査・規模条件が
     あるため、最初は AdSense 連携だけでよい。

3. **「Rewarded（リワード）」広告ユニットを作成**
   - GAM で広告ユニットを新規作成し、リワード用に設定。Web では**ユニットサイズは配信に影響しない**。
   - デフォルトは「1 回の視聴＝1 リワード」で、上書き可能。
   - 配信設定で **「Block non-instream video ads」を OFF** にする（ON だとリワード動画が出ない）。

4. **サイト側に Google Publisher Tag (GPT) を実装** ✅ 実装済み
   - `googletag` のリワード広告 API（`rewardedSlotReady` / `makeRewardedVisible` /
     `rewardedSlotGranted` / `rewardedSlotClosed`）を [rewardedAd.ts](../frontend/src/lib/rewardedAd.ts)
     にまとめ、[Map.tsx](../frontend/src/components/Map.tsx) の `openVideoReward` が
     「nonce 発行 → 広告表示 → granted で `POST /points/reward/video`」を行う。モックの
     10 秒カウントは廃止した。広告ユニットパスは `/23356418393/chizunurie_rewarded_video`
     （`NEXT_PUBLIC_REWARDED_AD_UNIT` で上書き可）。
   - **重要：Web の GPT リワードに SSV ポストバックは無い**。報酬付与は AdMob（アプリ）と違い
     クライアントの `rewardedSlotGranted` で判断する（GPT 公式仕様）。そのため GAM 広告ユニットの
     「サーバーサイドで認証されるポストバック URL」は**メディエーション（アプリ SDK 経由）専用**で、
     今回の Web 配信では発火しない。**空欄のままで正しい。**
   - 不正対策は**自前 nonce ＋サーバー側のクールダウン/日次上限**で行う（実装済み）：視聴開始時に
     `POST /points/reward/video/nonce` で 1 回限りの nonce を発行し、`rewardedSlotGranted` 後の
     請求でそれを照合・単回使用にする（[routes/points.ts](../backend/src/routes/points.ts) /
     [points.ts](../backend/src/lib/points.ts) の `issueVideoRewardNonce` / `claimVideoReward`）。
     ※ `user_points` に `reward_nonce` / `reward_nonce_at` 列を追加済み。
     `npm run db:push`（または `db:migrate`）でマイグレーション `0002_neat_maximus.sql` を適用すること。

5. **必須の周辺設定**
   - **`ads.txt`** をドメイン直下に設置（GAM / AdSense のパブリッシャー ID を記載）。
   - ページに `<meta name="viewport" content="width=device-width, initial-scale=1.0">`
     （リワードはモバイル最適化ページが対象。Next.js なら標準で入る）。
   - サイト審査・ポリシー準拠が前提。

## ポリシー上の必須要件（リワード特有）

- **opt-in 必須**：ユーザーが自分で「動画を見る」を選ぶ形（現状の ▶ ボタン方式で OK）。
- **報酬の事前明示**：「何をすると」「何が貰えるか」を視聴前に明確表示（現状のモーダル文言で対応済み）。
- **5 秒以上の表示**で初めてリワード付与。現状は 10 秒なので余裕で満たす。
- **報酬は自プラットフォーム内でのみ使えるもの**に限る（塗りポイントは OK。現金・換金性のあるものは✕）。

## 現実的な注意点

- **Web のリワードはフィル率・単価がアプリより低め**になりがち（在庫・需要がモバイルアプリほど
  潤沢でない）。最初は「埋まらないこともある」前提で、未充填時はボタンを一時的に隠す／トーストで
  案内する作りにしておくと安全。
- 将来リワード収益を本気で伸ばすなら、**PWA → アプリ化（TWA / Capacitor）して AdMob のリワード**を
  入れるのが王道（AdMob の方がリワード在庫が圧倒的に多い）。

## おすすめの進め方

1. まず **GAM 無料版に登録 → AdSense 連携 → リワードユニット作成**
2. GPT を実装し、現状のモックの完了処理と差し替え（サーバー検証付き）
3. `ads.txt`・審査を通す
4. 収益が見えてきたらアプリ化＋AdMob を検討
