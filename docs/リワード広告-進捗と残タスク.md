## いちばん重要な技術的結論：Web の GPT リワードに SSV は無い

- **アプリ（AdMob）**のリワードには、Google がサーバーへ署名付きで叩く **SSV ポストバック**がある。
- **Web の GPT リワードは報酬付与がクライアントイベント `rewardedSlotGranted` で完結**する（公式仕様）。
- したがって GAM 広告ユニットの **「サーバーサイドで認証されるポストバック URL」はメディエーション
  （アプリ SDK 経由）専用**で、今回の Web 配信では**発火しない**。→ **空欄のままで正しい。**
- 不正対策は **自前 nonce ＋ サーバー側のクールダウン/日次上限**で実装した（下記）。

## 開発環境でのテスト方法

dev ビルドは自動で **Google 公式テストユニット `/22639388115/rewarded_web_example`** を使う
（ドメイン審査・ads.txt 不要で必ずテスト広告がフィルする）。

```bash
# dev DB に nonce 列を反映（この DB は db:push 運用なので push を使う。後述の注意参照）
cd backend && node --env-file=.env ./node_modules/.bin/drizzle-kit push   # 適用済み
cd backend && npm run dev      # :3001
cd frontend && npm run dev     # :3000
```

1. `http://localhost:3000` で**ログイン**（ボタンはログイン中のみ・nonce API も認証必須）。
2. 左下「▶ 動画を見て回復」→ テスト広告が全画面表示 → 最後まで視聴で回復＋トースト。
3. 出ないとき：広告ブロッカーを無効化／DevTools のスマホ表示で再試行／クールダウンは
   `POST /points/debug/reset`（開発者のみ）でリセット可。

### 解決済み：ドメインの粒度（apex vs サブドメイン）✅

AdSense の登録サイトは **ルート `gamebox777.org`**。Google は `https://gamebox777.org/ads.txt`
（ルート直下）を見にいくため、apex 側でも ads.txt に到達させる必要があった。
→ **Cloudflare で apex の ads.txt をサブドメインへ 301 リダイレクト**して解決済み（2026-06-08）。
ads.txt クローラは1回のリダイレクトを追従するため問題ない。

Cloudflare 側で追加・変更したのは以下の2点のみ（`unitygamebox.com` には未着手）：

- apex に `AAAA 100::`（Proxied）レコードを1件追加（apex を Cloudflare 経由にするため）。
- Redirect Rule「apex ads.txt redirect」を1件作成・デプロイ
  （`gamebox777.org/ads.txt` → `https://chizunurie.gamebox777.org/ads.txt`・301）。

確認結果：
```
$ curl -sIL https://gamebox777.org/ads.txt
HTTP/2 301  (location: https://chizunurie.gamebox777.org/ads.txt, server: cloudflare)
HTTP/2 200  (content-type: text/plain, content-length: 59)
```

## 現実的な注意

- Web のリワードはフィル率・単価がアプリより低め。未充填時はトースト案内で逃がす実装にしてある
  （`unavailable` / `dismissed`）。
- 高負荷の不正対策は Web では限界があるが、**nonce（単回使用）＋ 30分クールダウン＋1日5回上限**で
  乱用は実用上の範囲に抑えている。
