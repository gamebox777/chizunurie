## ⚠️ 2026-06-11：本番で no fill（在庫なし）— いま調査中の問題

審査通過当日、本番（Web・札幌のユーザー）で「▶ 動画を見て回復」を押したが広告が出なかった。
操作ログ（video_reward の meta.debug.trail）：

```
0ms start → 128ms gpt_loaded → 216ms display() → 227ms slotRequested
→ 329ms slotResponseReceived → 330ms slotRenderEnded isEmpty=true
→ 8218ms ready_timeout → unavailable
```

### ログの読み方（コードは正常）

- `slotRequested`→`slotResponseReceived`→`isEmpty=true`：**GAM へのリクエストは届き、
  応答も返ってきた上で「配信できる広告ゼロ」＝純粋な no fill**。
  広告ブロッカー・実装バグ・回線の問題ではない（それらは `slotRenderEnded` 自体が出ない）。

### 原因の候補（可能性順）

1. **GAM「在庫 → サイト」に新ドメイン `unitygamebox.com` が未登録／未承認**（最有力）。
   AdSense の「サイト」承認と GAM の「在庫 → サイト」承認は**別物**で、後者が通っていないと
   AdSense 連携の需要は広告ユニットに流れない。ドメイン変更直後なので旧ドメインのままの可能性。
2. **承認直後で需要が立ち上がっていない**。ads.txt クロールが当日 10:40・no fill が 11:51。
   承認・クロール後、需要が流れ始めるまで **24〜48時間**かかるのが普通。
3. **Web リワードの需要が日本では構造的に薄い**（全部正しくてもフィル率は低め）。

### 検討した代替案：インタースティシャルへの変更 → 不可

GPT の `OutOfPageFormat.INTERSTITIAL` は存在するが、今の「ボタン→全画面→視聴完了で報酬」
フローには使えない：

- **手動で「今すぐ表示」する API が無い**（ページ遷移時に GPT が自動で挟む形式）。
- **`rewardedSlotGranted` 相当の「視聴完了」イベントが無い**（nonce 付与の根拠が消える）。
- Google 側の頻度キャップが強制され、繰り返し回復とも相性が悪い。

→ フィルが改善しない場合の本命は **H5 Games Ads（Ad Placement API・`adBreak({type:'reward'})`）**。
Web でも「ボタン→リワード動画→視聴完了で報酬」がそのまま組める。AdSense ベースで需要も
GPT Web リワードより厚い（H5 Games Ads の審査が別途必要）。

### 対応（2026-06-11 実装）：Web はディスプレイ広告のオーバーレイ表示へ切り替え

no fill が解消する見込みが立たないため、Web の「▶ 動画を見て回復」は
**AdSense ディスプレイユニット（`chizunurie_display_1`・slot `5435028180`）を
全画面オーバーレイで表示し、閉じた時点で報酬請求に進む**方式に変更した：

- [frontend/src/lib/displayAd.ts](../frontend/src/lib/displayAd.ts)（新規）
  `showDisplayAdOverlay(slot)`：オーバーレイ DOM を生成 → `<ins class="adsbygoogle">` に
  `push({})` → 閉じるボタンで `{ outcome:'granted', debug }` を返す。
  フィルの有無（`data-ad-status`）は報酬判定に使わず debug（`renderIsEmpty`・`trail`）に記録のみ。
- [frontend/src/components/Map.tsx](../frontend/src/components/Map.tsx)
  Web 分岐を `showRewardedAd(GPT)` → `showDisplayAdOverlay(DISPLAY_AD_SLOT)` に差し替え。
  スロットは `NEXT_PUBLIC_DISPLAY_AD_SLOT` ＞ 既定 `5435028180`。
  nonce 発行 → 表示 → granted → 請求 の流れ・操作ログ（video_reward）は従来のまま。
- adsbygoogle.js 本体は既存の `AdSenseLoader`（layout.tsx・Web のみ）が読み込み済み。
- `rewardedAd.ts`（GPT 版）はコードとして残置（GAM 在庫→サイト承認後に戻す選択肢用）。

> ⚠️ **ポリシーリスク（了承の上で実施）**：報酬と紐付けたディスプレイ広告表示は AdSense の
> incentivized traffic（報酬による広告表示誘導）に該当し得る。モーダル表示もインタースティシャル的
> 配置としてグレー。アカウント制限・停止の警告が来た場合は即座にこの方式を止めること。
> リワード需要を正規に使うなら H5 Games Ads（下記）が本筋。

### 見送った改善

- GPT 版の `slotRenderEnded isEmpty=true` 即時 `unavailable` 化（8秒待ちの解消）は、
  Web が GPT を使わなくなったため見送り（戻す場合は `rewardedAd.ts` の `onRenderEnded`）。

## いちばん重要な技術的結論：Web の GPT リワードに SSV は無い

- **アプリ（AdMob）**のリワードには、Google がサーバーへ署名付きで叩く **SSV ポストバック**がある。
- **Web の GPT リワードは報酬付与がクライアントイベント `rewardedSlotGranted` で完結**する（公式仕様）。
- したがって GAM 広告ユニットの **「サーバーサイドで認証されるポストバック URL」はメディエーション
  （アプリ SDK 経由）専用**で、今回の Web 配信では**発火しない**。→ **空欄のままで正しい。**
- 不正対策は **自前 nonce ＋ サーバー側の上限**で実装した（下記）。

3. 出ないとき：広告ブロッカーを無効化／DevTools のスマホ表示で再試行／クールダウンは
   `POST /points/debug/reset`（開発者のみ）でリセット可。
