# マネタイズ方針（広告・動画）

白地図塗りゲーム（chizunurie）を PC / スマホで広告・動画再生によりマネタイズするための検討メモ。

## 前提

- 本アプリは **ネイティブアプリではなく Web アプリ（Next.js）**。広告手段の選択肢に影響する。
- 画面は **全画面の地図（MapLibre GL JS）**。広告の置き場所が最大の課題。
- ゲーム性（メッシュを塗る・達成感を得る）があるため、**リワード（報酬）型動画広告と相性が良い**。

## 1. 主な手段と相性

| 手段 | 仕組み | 相性 | 備考 |
|---|---|---|---|
| Google AdSense | 自動でディスプレイ広告 | △ 全画面地図で置き場所が難しい | 導入が一番簡単。審査あり |
| Google Ad Manager (GAM) | 枠を自設計＋複数広告会社で入札 | ○ 枠を作り込めば収益最大化 | 運用が重い。規模が出てから |
| リワード動画広告 | 動画視聴で報酬付与 | ◎ ゲームと最も相性が良い | 本命（下記参照） |
| インタースティシャル | 画面切替時に全画面広告 | ○ 出しすぎ注意 | UX 悪化しやすい |
| アフィリエイト | 旅行・地図グッズ・ふるさと納税等 | ○ 「行った街」と文脈が合う | 楽天 / Amazon など |
| サブスク / 投げ銭 | 広告非表示・限定色など | ○ 補完的に | Stripe 等 |

## 2. 本命：リワード動画 ×「塗りゲーム」

「メッシュを塗って達成感を得る」ゲーム性を活かし、動画視聴と報酬を結びつける。

- 動画を見て **今日の塗り回数を回復**（1日の塗り上限を設け、動画で追加）
- 動画を見て **限定カラー / 特別な塗りパターンを解放**
- 動画を見て **塗った範囲を高解像度画像で保存・SNS シェア**
- **GPS 塗りのブースト**（一定時間まとめて塗れる等）

報酬を絡めると視聴率（＝収益）が上がり、「広告がうざい」になりにくい。

## 3. Web アプリとしての技術的な現実

- **AdMob は Web では使えない**（アプリ専用 SDK）。Web では AdSense / Google Ad Manager 系を使う。
- Web 向けリワード動画は選択肢が限られる。実務的には:
  - **Google Ad Manager の Rewarded ads（web）** を使う
  - もしくは将来 **PWA → アプリ化（Capacitor / TWA）** して AdMob のリワードを入れる
- 全画面地図なので、AdSense 自動広告より **自分で枠（下部バナー / サイドバー / モーダル内）を切る** 方が現実的。
- レスポンシブ出し分けの定番:
  - スマホ縦画面 … 下部固定バナー
  - PC … 地図横の余白にサイドバー

## 4. 現実的なロードマップ

1. **まず Google AdSense** で下部バナー 1 枠（最小コスト・審査通過の実績作り）
2. ユーザーが増えたら **Google Ad Manager** へ移行し、リワード動画＋枠最適化
3. **広告非表示サブスク**（月 100〜300 円程度）を併設し、広告嫌いの層からも回収
4. 規模が出たら **アフィリエイト**（旅行・ふるさと納税など「街」と相性の良いもの）を追加

## 5. 実装の足がかり（メモ）

- 広告枠: `frontend/src/components/Map.tsx` の左下パネル付近にレスポンシブな広告枠を 1 つ追加。
- リワード設計: 塗り上限（1日 N 回）＋動画視聴で回復、の仕組みを `painted` 管理まわりに追加。
- 広告非表示サブスク: Stripe + バックエンド（`backend/`）で課金状態を持たせ、フラグで広告を出し分け。
